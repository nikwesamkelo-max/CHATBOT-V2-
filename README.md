Multi Memory Chatbot v2 🤖

Overview

Multi Memory Chatbot v2 is a Python console chatbot that can remember multiple pieces of information about a user. Instead of storing only one value, the chatbot uses structured memory to organize different facts and respond based on what it knows.

This project was built to practice fundamental software engineering concepts such as memory management, classification, decision-making, and conversational flow.

---

Features

- Store multiple user facts (name, favorite game, favorite food, etc.)
- Recognize different ways of providing information
- Respond using stored memories
- Maintain conversation history
- Use classification and decision-making to determine actions
- Organize information using structured memory

---

Concepts Practiced

- Variables and Data Structures
- Functions
- Loops
- Dictionaries and Lists
- Input Processing
- Classification
- Decision Making
- Structured Memory
- Basic Conversational AI Architecture

---

Example

User:

«My name is Sam»

Bot:

«Nice to meet you, Sam. I'll remember your name.»

User:

«My favorite game is FIFA»

Bot:

«Got it! I'll remember that your favorite game is FIFA.»

User:

«What's my favorite game?»

Bot:

«Your favorite game is FIFA.»

---

Project Architecture

Input
↓
Classification
↓
Decision Making
↓
Memory Update or Retrieval
↓
Response

---

What I Learned

This project helped me understand that chatbots are not just a collection of print statements. They are systems that:

- Receive information
- Understand meaning
- Make decisions
- Store and retrieve information
- Generate responses

It also introduced me to thinking about software as interconnected systems rather than isolated lines of code.
